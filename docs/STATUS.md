# Поточний стан і дорожня карта

Чесний зріз: що **перевірено наскрізно**, що **реалізовано**, і що в **дорожній карті**. MVP свідомо побудовано фазами — архітектуру закладено під подальші фази без переробки.

## ✅ Працює наскрізно (перевірено на реальній 40-хвилинній зустрічі)

- **`docker compose up`** піднімає весь стек (FastAPI · Postgres · Redis · Qdrant); `/health` справний (статус кожного компонента).
- **Capture → транскрипт:** завантаження зустрічі з **consent-гейтом** → постановка в Redis-чергу → host-GPU воркер (**WhisperX `small` + pyannote-3.1**) → **діаризований транскрипт** `[{speaker, start, end, text}]` у Postgres.
- **Веб-інтерфейс (React SPA):** список проєктів, завантаження зустрічі, **статус обробки в реальному часі** (polling), перегляд діаризованого транскрипту зі спікерами й таймкодами.
- Результат прогону на тестовій зустрічі: **493 сегменти, 4 спікери**, мова uk, ~7.5 хв чистого inference.

## 🔧 Реалізовано (код готовий, у складі MVP)

- **Per-project памʼять (ingestion):** транскрипт → speaker/gap-aware чанки → embeddings **bge-m3** → **Qdrant з ізоляцією за `project_id`** (памʼять одного проєкту не бачить інші); сутності (**action items, рішення**) з цитатами → Postgres. Інжестія **ідемпотентна** (delete-then-write + детермінований `uuid5`-id точок).
- **Прототип резюме (Агент-2):** діаризований транскрипт → структуроване резюме з **grounding**, **цитатами на конкретні репліки `[#N]`** і **confidence → HITL-гейтом** (≥0.85 авто-апрув / 0.6–0.85 ревʼю / <0.6 reject). Локальна LLM (Ollama) або, опційно, OpenAI для «непублічних» зустрічей.

## 🗺️ Дорожня карта (наступні фази)

- **Запити до памʼяті проєкту:** гібридний retrieval (BM25 + dense) → cross-encoder reranker → `/ask` (SSE) з цитатами та **abstention** («не зафіксовано» замість вигадки).
- **Агент:** ReAct + tools (`search_memory`, Text-to-SQL по сутностях, `propose_*`), LangGraph із вбудованим HITL.
- **Дії за апрувом:** розсилка резюме учасникам + інтеграції **Jira / Slack / email** через чергу апрувів і ідемпотентний executor (механізм підтвердження — ядро; конектори нарощуються поступово).
- **Eval-гейт:** RAGAS (faithfulness / relevancy / abstention) + регресійний гейт; інтеграція STT-бенчмарку.

---

> Принципи, закладені від першої фази: **local-first** (приватність), **pluggable** (STT/LLM за конфігом), **propose-then-commit** (жоден side-effect без рішення людини). Деталі — [ARCHITECTURE.md](ARCHITECTURE.md).
