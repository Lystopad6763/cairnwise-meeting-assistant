# План на завтра — покращення та перевірка

Пріоритети: **P0** = зробити першим (найбільший важіль / розблоковує демо), **P1** = важливо, **P2** = nice-to-have.

## A. Перевірка (verification) — переконатися, що все справді працює

- [ ] **P0 · Наскрізний live-флоу в браузері.** Записати коротку зустріч → транскрипція → relabel спікерів → резюме **local** → резюме **cloud** → порівняти якість. Має пройти без помилок; зафіксувати скріншоти для звіту. (Передумова: запущені `worker.py` + `summary_worker.py`.)
- [ ] **P0 · Реальна крос-проєктна ізоляція.** Заінжестити запис orbit у проєкт `orbit`, тоді `check_isolation.py --real <orbit_id>` + `--real <acme_id>`: довести, що пошук в `acme` не бачить `orbit` на **реальних** даних (зараз дані лише в acme).
- [ ] **P1 · Перезапуск LLM-бенчмарку з нової локації.** `python eval/llm_extraction_benchmark/extraction_agent.py` — підтвердити самодостатні шляхи + що `neural-chat` лишається лідером локальних.
- [ ] **P1 · Регресія збірки.** `cd frontend && npm run build` (tsc+vite) зелено; бекенд `python -m py_compile app/**` зелено; `curl /health` = ok.
- [ ] **P2 · Перевірка воркерів/черг.** `redis-cli LLEN cairnwise:transcribe|summarize` = 0 після прогонів; жодних `failed` без причини.

## B. Покращення (improvement)

- [ ] **P0 · Якість резюме (echo-баг).** `neural-chat` відлунює few-shot приклад на довгому UA-тексті. Спробувати по черзі: (а) приклад із явно-плейсхолдерними токенами («УЧАСНИК_A», «‹ЗАДАЧА›») або **прибрати конкретний приклад**; (б) **стиснення контексту** — резюмувати чанками, тоді злити; (в) порівняти з cloud (`gpt-4o-mini`). Критерій: на m05 owner = реальне підписане імʼя, факти не з прикладу.
- [ ] **P1 · Сигнал «воркер живий».** Воркери пишуть Redis-ключ із TTL (heartbeat); `/health` віддає стан воркерів; UI показує банер «STT/Summary-воркер не запущено», коли черга не рухається. (Прибирає весь клас «застрягло в черзі».)
- [ ] **P1 · Авто-інжест після транскрипції.** STT-воркер після `transcribed` робить `LPUSH cairnwise:ingest`; підняти справжній `ingest_worker.py` (BRPOP) — памʼять будується автоматично, без ручного `ingest.py`.
- [ ] **P2 · Прослуховування запису в UI.** `<audio>`-плеєр на сторінці зустрічі (через новий `GET /meetings/{id}/audio`) — чути, що записалось, до/після транскрипції.
- [ ] **P2 · Cloud-резюме за замовчуванням для якості** (коли є ключ і зустріч «непублічна») — лишити local як приватний дефолт, але підказати перемкнутись.
